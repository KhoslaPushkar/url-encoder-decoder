Complete Guide to URL Encoding

Learn everything about URL encoding, from basic concepts to advanced implementation techniques across different programming languages.

Tutorial September 10, 2025 10 min read

What is URL Encoding?

URL encoding, also known as percent-encoding, is a mechanism for encoding information in a Uniform Resource Identifier (URI) under certain circumstances. It's used to represent characters that are not allowed in a URL or have special meaning, by replacing them with a "%" followed by two hexadecimal digits.

This encoding is necessary because URLs can only be sent over the Internet using the ASCII character set. Since URLs often contain characters outside the ASCII set, they must be converted into a valid ASCII format.

Why URL Encoding is Important

URL encoding serves several crucial purposes in web development:

  • Data Integrity: Ensures that special characters don't interfere with URL structure
  • Security: Prevents injection attacks by encoding potentially dangerous characters
  • Compatibility: Makes URLs portable across different systems and browsers
  • SEO: Creates clean, readable URLs that search engines can properly index

Note: URL encoding is defined in RFC 3986, which standardizes how URIs should be structured and encoded.

Characters That Need Encoding

Not all characters need to be encoded in a URL. The following categories of characters must be encoded:

Reserved Characters

These characters have special meaning in URLs and must be encoded when used for other purposes:

! * ' ( ) ; : @ & = + $ , / ? % # [ ]

Unsafe Characters

These characters can cause problems in URLs and should always be encoded:

space " < > % { } | \ ^ ~ ` [ ]

Non-ASCII Characters

Any character outside the ASCII character set must be encoded, including:

  • Accented characters (é, ñ, ü)
  • Characters from non-Latin scripts (中文, العربية, русский)
  • Emoji and other Unicode characters

URL Encoding in Different Programming Languages

JavaScript

JavaScript provides two main functions for URL encoding:

// encodeURIComponent - encodes everything except: A-Z a-z 0-9 - _ . ! ~ * ' ( ) const encoded = encodeURIComponent("Hello World! &?=+"); console.log(encoded); // Hello%20World!%20%26%3F%3D%2B // encodeURI - encodes everything except: A-Z a-z 0-9 ; , / ? : @ & = + $ - _ . ! ~ * ' ( ) # const encodedURI = encodeURI("https://example.com/hello world"); console.log(encodedURI); // https://example.com/hello%20world // Decoding const decoded = decodeURIComponent(encoded); console.log(decoded); // Hello World! &?=+

Warning: encodeURI() should not be used to encode entire URLs, as it doesn't encode characters that have special meaning in URLs like &, ?, =, etc. Use encodeURIComponent() for query parameters.

Python

Python's urllib.parse module provides URL encoding functions:

from urllib.parse import quote, quote_plus, unquote # Basic encoding encoded = quote("Hello World! &?=+") print(encoded) # Hello%20World%21%20%26%3F%3D%2B # Encoding with plus signs for spaces (common in form data) encoded_plus = quote_plus("Hello World! &?=+") print(encoded_plus) # Hello+World%21+%26%3F%3D%2B # Decoding decoded = unquote(encoded) print(decoded) # Hello World! &?=+

PHP

PHP offers several functions for URL encoding:

// Basic encoding $encoded = urlencode("Hello World! &?=+"); echo $encoded; // Hello+World%21+%26%3F%3D%2B // Raw encoding (doesn't convert spaces to +) $raw_encoded = rawurlencode("Hello World! &?=+"); echo $raw_encoded; // Hello%20World%21%20%26%3F%3D%2B // Decoding $decoded = urldecode($encoded); echo $decoded; // Hello World! &?=+ $raw_decoded = rawurldecode($raw_encoded); echo $raw_decoded; // Hello World! &?=+

Best Practices for URL Encoding

1. Encode Individual Components, Not Entire URLs

Always encode URL components separately rather than encoding the entire URL:

// Good practice - encode components separately const baseUrl = "https://example.com/search"; const query = "hello world & special chars"; const encodedQuery = encodeURIComponent(query); const fullUrl = `${baseUrl}?q=${encodedQuery}`; // Result: https://example.com/search?q=hello%20world%20%26%20special%20chars // Bad practice - encoding the entire URL const badUrl = encodeURI("https://example.com/search?q=hello world & special chars"); // Result: https://example.com/search?q=hello%20world%20&%20special%20chars // The & character is not encoded, which breaks the URL

2. Use Appropriate Encoding for Different URL Parts

Different parts of a URL have different encoding requirements:

  • Path segments: Use standard percent-encoding
  • Query parameters: Encode both parameter names and values
  • Fragment identifiers: Encode special characters

3. Handle International Characters Properly

For non-ASCII characters, ensure proper Unicode handling:

// JavaScript example with Unicode characters const text = "café résumé"; const encoded = encodeURIComponent(text); console.log(encoded); // caf%C3%A9%20r%C3%A9sum%C3%A9 // Python example with Unicode from urllib.parse import quote text = "café résumé" encoded = quote(text) print(encoded) # caf%C3%A9%20r%C3%A9sum%C3%A9

Common URL Encoding Mistakes

1. Double Encoding

This occurs when already-encoded text is encoded again:

// Original text: hello world // First encoding: hello%20world // Second encoding: hello%2520world (the % is encoded as %25) // To avoid this, check if text is already encoded before encoding function safeEncodeURIComponent(str) { try { // Try to decode first - if it changes, it was already encoded const decoded = decodeURIComponent(str); if (decoded !== str) { // String was already encoded, return as-is return str; } } catch (e) { // decoding failed, probably not encoded } return encodeURIComponent(str); }

2. Incorrect Character Set Handling

Ensure you're using the correct character encoding (usually UTF-8):

// PHP example - setting proper charset ini_set('default_charset', 'UTF-8'); // Python example - explicit encoding from urllib.parse import quote text = "café" encoded = quote(text.encode('utf-8'))
