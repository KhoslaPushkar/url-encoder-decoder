Complete Guide to URL Encoding

Learn everything about URL encoding, from basic concepts to advanced implementation techniques across different programming languages.

Tutorial September 10, 2025 10 min read

What is URL Encoding?

If you’ve ever shared a link that included spaces or strange-looking characters, you’ve already bumped into the need for URL encoding. In simple terms, URL encoding—also called percent-encoding—is a way of making sure web addresses only contain safe, valid characters. Any character that’s not allowed in a URL is replaced with a % followed by two hexadecimal digits. For example, a space becomes %20.

Why do we need this? Because the internet relies on the ASCII character set, which doesn’t cover accented letters, emojis, or many symbols we use every day. Without encoding, a browser wouldn’t know how to process those characters, and the link could break.

Why URL Encoding Matters

It’s not just a technical detail; encoding is critical for a few key reasons:

  • Data integrity: Prevents special characters from messing up the URL’s structure.
  • Security: Blocks malicious input that could otherwise be used in injection attacks.
  • Compatibility: Ensures links work across different browsers, systems, and platforms.
  • SEO: Clean, properly encoded URLs are easier for search engines to crawl and index.

Note: The official rules for URL encoding are defined in RFC 3986, which lays out how URIs should be structured.

Characters That Require Encoding

Not every character needs to be encoded, but some definitely do. Here’s the breakdown:

Reserved Characters

These symbols have special meaning in URLs. If you use them for anything else, they need to be encoded:

! * ' ( ) ; : @ & = + $ , / ? % # [ ]

Unsafe Characters

These can cause all sorts of trouble in a URL and should always be encoded:

space " < > % { } | \ ^ ~ ` [ ]

Non-ASCII Characters

Any character outside the standard ASCII set—think accented letters, non-Latin scripts, or even emojis—must be encoded. Examples include:

  • Accents: é, ñ, ü
  • Other scripts: 中文, العربية, русский
  • Emoji: 😀 🎉 🚀

Encoding in Different Programming Languages

JavaScript

JavaScript gives you two main functions: encodeURIComponent and encodeURI. The first one is best for encoding query parameters, while the second handles full URLs but leaves some characters untouched. 

// encodeURIComponent - encodes almost everything
const encoded = encodeURIComponent("Hello World! &?=+");
console.log(encoded); // Hello%20World!%20%26%3F%3D%2B

// encodeURI - better for whole URLs
const encodedURI = encodeURI("https://example.com/hello world");
console.log(encodedURI); // https://example.com/hello%20world

// Decoding
const decoded = decodeURIComponent(encoded);
console.log(decoded); // Hello World! &?=+

Warning: Don’t use encodeURI() for query parameters— it skips &, ?, and =, which can break your link. Stick with encodeURIComponent() for those.

Python

In Python, the urllib.parse module has what you need. quote and quote_plus are the main players: 

from urllib.parse import quote, quote_plus, unquote

# Basic encoding
encoded = quote("Hello World! &?=+")
print(encoded) # Hello%20World%21%20%26%3F%3D%2B

# Form-style encoding (spaces as +)
encoded_plus = quote_plus("Hello World! &?=+")
print(encoded_plus) # Hello+World%21+%26%3F%3D%2B

# Decoding
decoded = unquote(encoded)
print(decoded) # Hello World! &?=+

PHP

PHP developers have a couple of options—urlencode() and rawurlencode()—depending on whether you want spaces as + or %20: 

// urlencode - turns spaces into +
$encoded = urlencode("Hello World! &?=+");
echo $encoded; // Hello+World%21+%26%3F%3D%2B

// rawurlencode - uses %20 for spaces
$raw_encoded = rawurlencode("Hello World! &?=+");
echo $raw_encoded; // Hello%20World%21%20%26%3F%3D%2B

// Decoding
$decoded = urldecode($encoded);
echo $decoded; // Hello World! &?=+

Best Practices

1. Encode Components Separately

Don’t just run encodeURI() on an entire link. Always encode individual parts like query parameters to avoid broken URLs.

2. Match Encoding to Context

Use percent-encoding for paths, encode both names and values in query strings, and don’t forget fragments if you’re working with them.

3. Handle Unicode Properly

When dealing with characters outside ASCII, always ensure you’re encoding with UTF-8 to avoid corruption.

Common Mistakes

1. Double Encoding

This happens when something is encoded twice, leading to messes like %2520 instead of %20. Always check if your string is already encoded before applying it again.

2. Wrong Charset

Stick with UTF-8 unless you have a very specific reason not to. Otherwise, special characters like café or résumé might end up mangled.
